Common Mistakes & Solutions

Modifying List While Iterating

One of the most common mistakes is modifying a list while iterating over it, which can cause elements to be skipped.

❌ Wrong Way
for item in my_list: if condition: my_list.remove(item) # Dangerous!
✅ Correct Way
# Option 1: Iterate over copy for item in my_list[:]: if condition: my_list.remove(item) # Option 2: List comprehension my_list = [item for item in my_list if not condition]

Mutable Default Arguments

Using mutable objects as default arguments can lead to unexpected behavior.

❌ Wrong Way
def append_to_list(item, target_list=[]): target_list.append(item) return target_list
✅ Correct Way
def append_to_list(item, target_list=None): if target_list is None: target_list = [] target_list.append(item) return target_list
